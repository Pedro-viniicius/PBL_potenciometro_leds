Revisão de eletrônica

O que é um LED? 💡

LED significa Diodo Emissor de Luz (do inglês Light Emitting Diode). É uma pecinha que acende quando a corrente passa por ela no sentido certo.

⚠️
O LED tem lado certo! Ele só acende em um sentido.
  • Perna longa = positivo (+) → chamada de ânodo
  • Perna curta = negativo (−) → chamada de cátodo, vai para o GND
🧠
Dica para lembrar: a perna curta é a do lado cortado/chanfrado do LED, e vai para o lado negativo (GND).
🔋

Como ligar

A corrente entra pela perna longa (+), atravessa o LED e sai pela perna curta (−) em direção ao GND. No caminho, sempre colocamos um resistor.

5V → [resistor] → ➕ LED ➖ → GND

Revisão de eletrônica

Por que o LED precisa de resistor? 🛡️

🔥
Sem resistor, passa corrente demais pelo LED. Ele esquenta, brilha forte por um instante e queima.
🚰

Analogia

O resistor é como uma torneira meio fechada: deixa passar só um pouquinho de "água" (corrente), na medida certa.

📏

Função

O resistor limita a corrente que passa pelo LED, protegendo o componente.

🔢

Valor típico

Para um LED comum em 5V usamos um resistor de 220 Ω a 330 Ω.

✅
Regra de ouro: todo LED deste projeto vai ter sempre um resistor em série. São 8 LEDs → 8 resistores.
Revisão de eletrônica

Corrente, tensão, 5V e GND ⚡

🚰
Analogia do encanamento: imagine a eletricidade como água em um cano.

🌊 Corrente (A)

É a quantidade de carga que passa pelo fio. Como a água que corre dentro do cano. Mais corrente = mais "água passando".

⛰️ Tensão (V)

É a "força" / pressão que empurra a corrente. Como a pressão da água. Sem tensão, a corrente não anda.

🔴 5V

É o pino que fornece a tensão positiva (a "fonte de pressão") para o circuito.

⚫ GND (terra)

É o ponto de referência negativo (0 V). É para onde a corrente volta para fechar o circuito.

🧠
Pense em 5V e GND como as duas margens de um rio: a água (corrente) só corre porque uma margem é "mais alta" (5V) do que a outra (GND).
Revisão de eletrônica

Por que o circuito precisa ser fechado? 🔁

A corrente só passa se houver um caminho completo, de ida e volta: ela sai do 5V, atravessa os componentes e volta ao GND.

🚧
Se o caminho estiver interrompido (fio solto, perna fora do furo), a corrente não passa e o LED não acende.
✅
Circuito fechado = caminho completo = corrente passando = LED aceso!
🔄

O caminho da corrente

① Sai do 5V
② Passa pelo resistor
③ Atravessa o LED
④ Volta para o GND ⚫
🔁 ... e o ciclo se mantém

Conhecendo o Arduino

O que é o Arduino Uno? 🟢

É uma placa de prototipagem: um pequeno computador que conseguimos programar para ler sensores e controlar componentes (LEDs, motores...).

🧠

Cérebro

Tem um microcontrolador que executa o programa que escrevemos no computador.

🔌

Pinos

Pelos pinos ele se conecta ao mundo: recebe sinais (entrada) e envia sinais (saída).

🔋

Energia

É alimentado pelo cabo USB do computador e fornece 5V e GND para o circuito.

💬
Resumindo: nós escrevemos o programa → enviamos para a placa pelo cabo USB → a placa obedece e controla o circuito.
Conhecendo o Arduino

Os pinos do Arduino Uno 🔌

ARDUINO UNO

Entradas analógicas (A0 a A5) — usaremos A0 para o potenciômetro:

🔢 Pinos digitais

São 14 (de 0 a 13). Trabalham com 2 estados: ligado ou desligado.

📈 Entradas analógicas

São 6 (A0 a A5). Leem valores variáveis, não só ligado/desligado.

🔋 5V e GND

5V alimenta o circuito; os pinos GND fecham o caminho da corrente.

Conhecendo o Arduino

Entrada × Saída · Digital × Analógico 🔁

➡️ Entrada vs. ⬅️ Saída

Entrada: a placa recebe / lê informação do mundo (ex.: ler o potenciômetro).

Saída: a placa envia / controla algo no mundo (ex.: acender um LED).

🧠
Pense na placa como uma pessoa: a entrada é o que ela vê/ouve; a saída é o que ela faz/fala.

🔢 Digital vs. 📈 Analógico

Sinal digital: só 2 valores → ligado (HIGH) ou desligado (LOW). Como um interruptor de luz.

Sinal analógico: muitos valores entre o mínimo e o máximo. Como um dimmer ou o volume do som.

💡
O LED usamos como saída digital. O potenciômetro usamos como entrada analógica.
Componente novo

Componente novo: o potenciômetro 🎛️

É um resistor variável com um botão giratório.

🔊
Analogia simples: é igual ao botão de volume de um som antigo. Girou para um lado, aumenta; para o outro, diminui.

Fisicamente, ao girar o botão movemos um "cursor" sobre uma trilha resistiva por dentro. Isso muda a resistência e, com isso, muda a tensão que entregamos ao Arduino. O Arduino lê essa tensão como um número.

Componente novo

Por que o potenciômetro tem 3 terminais? 🔌

Ele funciona como um divisor de tensão: as duas pontas recebem a energia e o meio entrega o valor.

🔴

Terminal lateral 1

Vai para o 5V da placa.

🟠

Terminal do MEIO

Vai para a entrada analógica A0. É o "cursor" que entrega o valor lido.

⚫

Terminal lateral 2

Vai para o GND da placa.

✅
Resumo da ligação: lateral → 5V · meio → A0 · outra lateral → GND
Componente novo

O que acontece ao girar o botão? 🔄

Por dentro existe um cursor que desliza entre o lado do 5V e o lado do GND:

  • Cursor perto do GND → tensão no meio fica baixa (perto de 0 V)
  • Cursor no meio → tensão fica na metade (perto de 2,5 V)
  • Cursor perto do 5V → tensão fica alta (perto de 5 V)
📈
Ou seja: girar o botão = escolher uma tensão entre 0 V e 5 V para mandar ao Arduino.

🔢 Por que de 0 a 1023?

O Arduino transforma a tensão (de 0 V a 5 V) em um número inteiro usando um conversor de 10 bits.

10 bits conseguem representar 1024 valores diferentes: de 0 até 1023.

TensãoanalogRead()
0 V0
2,5 V≈ 512
5 V1023
Componente novo

Erros comuns ao ligar o potenciômetro ⚠️

🔀

Trocar o meio com a lateral

Se o fio de A0 não estiver no terminal do meio, a leitura fica estranha ou travada.

⚡

5V e GND trocados

Não queima, mas o valor fica invertido (gira para um lado e o número diminui em vez de aumentar).

🔌

Usar pino digital

O potenciômetro é analógico! O meio precisa ir para uma entrada A0–A5, não para um pino digital.

🧩

Pernas no furo errado

Na protoboard, os 3 terminais precisam estar em colunas diferentes, sem se encostar.

Conceitos de programação

O que é uma variável? 📦

Uma variável é uma caixinha com nome onde guardamos um valor para usar depois.

📦
Pense em uma caixa etiquetada: a etiqueta é o nome (ex.: leitura) e dentro tem um valor (ex.: 512).

Por que usamos variáveis?

  • Para guardar o que lemos do potenciômetro
  • Para reaproveitar o valor em vários lugares
  • Para deixar o código mais fácil de ler

int = número inteiro

Usamos int para guardar números inteiros (sem vírgula), como 0, 7, 512, 1023.

int leitura;          // cria a caixinha
leitura = 512;        // guarda 512 dentro
int nivel = 0;       // cria e já guarda 0
Conceitos de programação

void setup() e void loop() 🔁

Todo programa de Arduino tem essas duas partes obrigatórias.

⚙️ void setup()

Roda UMA única vez, quando a placa liga. Serve para preparar / configurar as coisas (ex.: dizer quais pinos são saída).

🧠
É como arrumar a mesa antes de começar a trabalhar.

🔄 void loop()

Roda SEM PARAR, repetindo para sempre. É onde fica a ação principal (ler o potenciômetro e acender os LEDs).

🧠
É como uma tarefa que você refaz o tempo todo, milhares de vezes por segundo.
void setup() {
  // roda 1 vez: preparação
}
void loop() {
  // roda para sempre: a ação repetida
}
Conceitos de programação

Configurar e controlar a saída digital 💡

pinMode()

Diz para a placa como usar um pino: como saída (OUTPUT) ou entrada (INPUT).

OUTPUT

Significa saída: o pino vai enviar sinal (acender o LED).

digitalWrite()

Liga ou desliga um pino de saída usando HIGH ou LOW.

🔌
HIGH = manda 5V no pino → LED acende.  ·  LOW = manda 0V no pino → LED apaga.
pinMode(2, OUTPUT);      // o pino 2 será uma SAÍDA
digitalWrite(2, HIGH);  // acende o LED do pino 2
digitalWrite(2, LOW);   // apaga o LED do pino 2
Conceitos de programação

Ler a entrada analógica: analogRead() 📈

analogRead(A0) lê a tensão no pino A0 e devolve um número de 0 a 1023.

Guardamos esse número em uma variável para usar depois:

int leitura = analogRead(A0);
// leitura agora vale algo entre 0 e 1023
💬
Essa é a parte de leitura e entrada: a placa está "ouvindo" o potenciômetro.

Do número aos LEDs

Precisamos transformar 0–1023 em "quantos LEDs acender" (0 a 8). Usamos a função map():

int nivel = map(leitura, 0, 1023, 0, 8);
// converte a faixa 0..1023 para 0..8

Agora nivel diz quantos LEDs devem ficar acesos.

Conceitos de programação

Decisão (if/else) e repetição (for) 🔀🔁

🔀 if / else

Faz a placa tomar decisão: SE uma condição for verdadeira, faça uma coisa; SENÃO, faça outra.

if (i < nivel) {
  digitalWrite(pino, HIGH); // acende
} else {
  digitalWrite(pino, LOW);  // apaga
}

🔁 for

Repete um bloco várias vezes. Em vez de escrever 8 linhas iguais, escrevemos uma que roda 8 vezes.

for (int i = 0; i < 8; i++) {
  // roda para i = 0,1,2,...,7
}
🧠
A repetição evita copiar o mesmo código 8 vezes — menos erro e mais fácil de ler!

Programação

Construindo o código por partes 🧩

1) Guardar os pinos dos LEDs (array)

// um "array" é uma lista de valores
int leds[] = {2,3,4,5,6,7,8,9};

2) Configurar como saída no setup

for (int i=0; i<8; i++) {
  pinMode(leds[i], OUTPUT);
}

3) Ler e converter no loop

int leitura = analogRead(A0);
int nivel = map(leitura,0,1023,0,8);

4) Acender progressivamente

for (int i=0; i<8; i++) {
  if (i < nivel)
    digitalWrite(leds[i], HIGH);
  else
    digitalWrite(leds[i], LOW);
}
🧠
A ideia central: se a posição do LED (i) for menor que o nível, acende; senão, apaga. Assim os LEDs acendem em sequência!
Programação

Código final completo ✅

// ============================================================
// 8 LEDs controlados por um potenciometro
// Gira o potenciometro -> acende mais ou menos LEDs em sequencia
// ============================================================

// Lista (array) com os 8 pinos digitais dos LEDs
int leds[] = {2, 3, 4, 5, 6, 7, 8, 9};
int totalLeds = 8;          // quantos LEDs temos
int pinoPot = A0;            // potenciometro no pino analogico A0

void setup() {
  Serial.begin(9600);       // abre o Monitor Serial p/ ver os valores
  // configura cada pino de LED como SAIDA
  for (int i = 0; i < totalLeds; i++) {
    pinMode(leds[i], OUTPUT);
  }
}

void loop() {
  // 1) LER a entrada analogica (0 a 1023)
  int leitura = analogRead(pinoPot);

  // 2) CONVERTER a leitura em quantidade de LEDs (0 a 8)
  int nivel = map(leitura, 0, 1023, 0, totalLeds);

  // 3) mostrar no Monitor Serial (ajuda a depurar)
  Serial.print("Leitura: ");
  Serial.print(leitura);
  Serial.print("  -> LEDs acesos: ");
  Serial.println(nivel);

  // 4) ACENDER os LEDs progressivamente
  for (int i = 0; i < totalLeds; i++) {
    if (i < nivel) {
      digitalWrite(leds[i], HIGH);  // dentro do nivel -> acende
    } else {
      digitalWrite(leds[i], LOW);   // fora do nivel  -> apaga
    }
  }

  delay(50);  // pequena pausa para estabilizar a leitura
}
